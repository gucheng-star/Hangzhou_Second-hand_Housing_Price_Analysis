# 杭州二手房总价预测 — 实验报告

> 数据集：杭州二手房 24951 条有效记录 | 训练:测试 = 8:2 | random_state=85

---

## 一、实验设置

### 1.1 数据预处理

所有实验使用相同的预处理数据 `预处理_杭州二手房信息汇总.xlsx`（19 列）：

| 特征类型 | 列名 | 处理方式 |
|----------|------|----------|
| 数值特征 | 建筑面积、关注度 | 手动转为 float |
| 房屋户型 | 室数、厅数、厨数、卫数 | 正则拆分 `X室Y厅Z厨W卫` |
| 所在楼层 | 楼层类别(0=低,1=中,2=高,3=未知)、总楼层数 | 正则拆分 + 标签编码 |
| 建筑类型 | 建筑类型_编码(0-4) | 标签编码 |
| 房屋朝向 | 房屋朝向_编码(0-95) | 标签编码 |
| 建筑结构 | 建筑结构_编码(0-6) | 标签编码 |
| 装修情况 | 装修情况_编码(0=毛坯,1=简装,2=精装) | 有序标签编码 |
| 文本字段 | 区域位置、核心卖点、小区名称、来源文件、来源工作表 | 部分用于目标编码 |

方法 A 和 C 额外对"区域位置"做目标编码，拆分为 `区_目标编码`+`板块_目标编码`（5 折交叉验证防泄漏）。

## 二、实验结果总览

| 方法 | R² | RMSE | MAE | 训练耗时 | 特征数 | 模型类型 |
|------|-----|------|-----|---------|--------|---------|
| **基线** | **0.5000** | 226.98 万 | 141.09 万 | <0.1s | 11 | 线性回归 |
| **方法 A** | **0.6432** | 191.72 万 | 114.44 万 | 0.02s | 13 | 线性回归 + 区域TE |
| **方法 B** | **0.6368** | 193.46 万 | 111.47 万 | 0.55s | 11 | 随机森林 |
| **方法 C** | **0.8458** | 126.05 万 | 64.28 万 | 4.01s | 13 | GBDT + 区域TE |

**提升幅度**：

```
基线 ────────────────────────── R² = 0.5000
  │
  ├── 方法 A (LR + 区域TE) ──── R² = 0.6432   ↑ 28.6%
  ├── 方法 B (随机森林)   ──── R² = 0.6368   ↑ 27.4%
  │
  └── 方法 C (GBDT + 区域TE) ─ R² = 0.8458   ↑ 69.2%  ★ 最优
```

## 三、各方法详细结果

### 基线：线性回归（11 特征）

**代码**：[多元回归分析.py](多元回归/多元回归分析.py)

**特征**：建筑面积、室数、厅数、厨数、卫数、楼层类别、总楼层数、建筑类型_编码、房屋朝向_编码、建筑结构_编码、装修情况_编码

| 指标 | 值 |
|------|-----|
| R² | 0.5000 |
| RMSE | 226.98 万元 |
| MAE | 141.09 万元 |
| 截距 | -147.40 |

**回归系数**（按绝对值降序）：

| 排名 | 特征 | 系数 | 解释 |
|------|------|------|------|
| 1 | 装修情况_编码 | +53.99 | 装修每提升一档 +54 万 |
| 2 | 厨数 | +43.46 | 每多 1 厨 +43 万 |
| 3 | 厅数 | -36.95 | 与面积共线导致符号异常 |
| 4 | 卫数 | +29.74 | 每多 1 卫 +30 万 |
| 5 | 建筑类型_编码 | -28.31 | — |
| 6 | 楼层类别 | -13.42 | 高层总价略低 |
| 7 | 室数 | +11.28 | 每多 1 室 +11 万 |
| 8 | 建筑结构_编码 | -4.63 | — |
| 9 | 建筑面积 | +3.56 | 每 +1㎡ 仅 +3.6 万（偏小，受共线性影响） |
| 10 | 总楼层数 | +2.46 | — |
| 11 | 房屋朝向_编码 | -0.84 | 几乎无影响 |

**分析**：
- 建筑面积系数仅 3.56，与实际行情（杭州均价约 3~5 万/㎡）严重不符，原因是被室数/厅数/厨数/卫数分摊了预测贡献（多重共线性）
- 厅数系数为负（不合理），同样是共线性导致
- 房屋朝向几乎无预测能力

### 方法 A：线性回归 + 区域位置目标编码

**代码**：[方法A_线性回归+区域目标编码.py](多元回归/方法A_线性回归+区域目标编码.py)

**思路**：在基线 11 特征基础上，加入用目标编码处理后的区域位置信息（区 + 板块）。

| 指标 | 值 |
|------|-----|
| R² | **0.6432** |
| RMSE | 191.72 万元 |
| MAE | 114.44 万元 |
| 截距 | -592.25 |
| 训练耗时 | 0.02s |
| 特征数 | 13 |

**回归系数**（按绝对值降序）：

| 排名 | 特征 | 系数 |
|------|------|------|
| 1 | 厨数 | +35.18 |
| 2 | 卫数 | +32.97 |
| 3 | 装修情况_编码 | +26.52 |
| 4 | 建筑类型_编码 | -24.13 |
| 5 | 厅数 | -14.08 |
| 6 | 楼层类别 | -13.32 |
| 7 | 室数 | +13.00 |
| 8 | 建筑面积 | +3.38 |
| 9 | 总楼层数 | +2.90 |
| 10 | 建筑结构_编码 | +2.59 |
| 11 | 房屋朝向_编码 | -0.89 |
| 12 | 板块_目标编码 | +0.67 |
| 13 | 区_目标编码 | +0.52 |

**关键发现**：
- R² 从 0.50 → 0.64，+14.3 个百分点，提升来自区域信息的注入
- 区_目标编码 和 板块_目标编码 的系数虽小（0.52 和 0.67），但因其编码值本身在 100~800 范围，实际贡献 = 0.5×500 ≈ 250 万量级
- 截距从 -147 变为 -592，因为区域编码携带了基础价格信息
- 线性模型的可解释性优势保留：每个特征的系数有明确的业务含义

### 方法 B：随机森林（300 棵树）

**代码**：[方法B_随机森林.py](多元回归/方法B_随机森林.py)

**思路**：用集成学习 Bagging 方法替代线性回归，300 棵树并行训练、投票取平均。

**超参数**：n_estimators=300, max_depth=15, min_samples_split=10

| 指标 | 值 |
|------|-----|
| R² | **0.6368** |
| RMSE | 193.46 万元 |
| MAE | 111.47 万元 |
| 训练耗时 | 0.55s |
| 特征数 | 11 |

**特征重要性**（按降序）：

| 排名 | 特征 | 重要性 |
|------|------|--------|
| 1 | 建筑面积 | 0.6970 |
| 2 | 总楼层数 | 0.1195 |
| 3 | 装修情况_编码 | 0.0407 |
| 4 | 卫数 | 0.0230 |
| 5 | 建筑结构_编码 | 0.0228 |
| 6 | 建筑类型_编码 | 0.0223 |
| 7 | 室数 | 0.0216 |
| 8 | 房屋朝向_编码 | 0.0182 |
| 9 | 楼层类别 | 0.0145 |
| 10 | 厅数 | 0.0117 |
| 11 | 厨数 | 0.0086 |

**关键发现**：
- 建筑面积的重要性高达 69.7%，压倒性第一——树模型通过非线性分叉自然解决了共线性问题
- 总楼层数意外地排第二（11.95%），在线性回归中它仅排倒数第二——树模型捕捉到了总楼层数与其他特征的非线性交互
- R² 0.6368 略低于方法 A（0.6432），但 MAE 更好（111 vs 114 万）——随机森林对中低价房源估得更准
- max_depth=15 的约束使 R² 低于无约束版本（0.67），但更好地控制过拟合

### 方法 C：GBDT + 区域位置目标编码（最优）

**代码**：[方法C_GBDT+区域目标编码.py](多元回归/方法C_GBDT+区域目标编码.py)

**思路**：将 Boosting 集成方法与区域目标编码结合。GBDT 逐棵修正残差捕获非线性关系，区域编码提供空间价格锚点。

**超参数**：n_estimators=200, max_depth=5, learning_rate=0.1, subsample=0.8

| 过拟合程度 | 0.0797 |

| MAE | 64.28 万元 |
| 训练耗时 | 4.01s |
| 特征数 | 13 |

**特征重要性**（按降序）：

| 排名 | 特征 | 重要性 |
|------|------|--------|
| 1 | 建筑面积 | 0.5527 |
| 2 | 板块_目标编码 | 0.1491 |
| 3 | 区_目标编码 | 0.1477 |
| 4 | 总楼层数 | 0.0758 |
| 5 | 卫数 | 0.0190 |
| 6 | 室数 | 0.0159 |
| 7 | 装修情况_编码 | 0.0095 |
| 8 | 厨数 | 0.0067 |
| 9 | 建筑结构_编码 | 0.0057 |
| 10 | 建筑类型_编码 | 0.0051 |
| 11 | 厅数 | 0.0050 |
| 12 | 楼层类别 | 0.0041 |
| 13 | 房屋朝向_编码 | 0.0035 |

**关键发现**：
- R² **0.8458** 是四个方案中最高，MAE 仅 64 万，预测精度大幅领先
- 区域信息（区+板块）合计重要性 29.7%，加上建筑面积（55.3%）共计 85%，三大特征主导预测
- 训练集 R²=0.9255 vs 测试集 R²=0.8458，存在约 8% 的过拟合——可接受范围内
- 学习率 0.1 + subsample 0.8 的策略有效控制了过拟合
- GBDT 的串行训练耗时 4s，约为随机森林（0.55s）的 7 倍，但在 2.5 万条数据上仍可接受

## 五、结论与建议

### 最优方案：方法 C（GBDT + 区域目标编码）

```
R²  = 0.8458  （解释 84.6% 的总价方差）
MAE = 64.28 万（平均预测误差仅 64 万）
```

相比基线的 0.50，提升 69.2%；相比方法 A 的 0.64，提升 31.5%。

### 如果追求可解释性：方法 A（线性回归 + 区域TE）
每个系数都有明确的业务含义（"多一个卫生间 +33 万"），R²=0.64 已能覆盖主要使用场景。

### 后续优化方向
1. 尝试 XGBoost / LightGBM（比 sklearn GBDT 更快更准）
2. 对区域做更细粒度的目标编码（如加入交叉特征：区×板块）
3. 删除房屋朝向特征（所有模型一致认为它最不重要）
4. 调参优化 GBDT 的 learning_rate 和 n_estimators 组合
5. 加入核心卖点的 NLP 特征（如"地铁房""学区房"等关键词）
